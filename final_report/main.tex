\renewcommand{\abstractname}{\kaishu \textbf{摘要}}
\begin{abstract}
近似最近邻搜索（Approximate Nearest Neighbor Search, ANNS）是向量检索、推荐系统和检索增强生成系统中的基础计算。其完整系统通常包含离线预处理与索引构建、在线候选搜索、Top-k 维护和 recall 评估等环节。本文围绕 DEEP100K 数据集实现并比较 Flat Scan、IVF-Flat、PQ-SIMD 和 HNSW 等检索路径，并从 SIMD、OpenMP 和 CUDA GPU 三个并行层次分析延迟与召回率的关系。实验表明，NEON 可将 96 维内积计算延迟由 20851.90 us 降至 7742.58 us；OpenMP 与 NEON 结合后，8 线程 Flat Scan 延迟降至 747.76 us，相比串行基线加速 27.89 倍；PQ-SIMD 在 Top-p=2000 时达到 0.99630 recall，并获得 7.37 倍加速。进一步的索引实验显示，在同一 DEEP100K 子集上，IVF-Flat 在 nprobe=16 时达到 0.9740 recall@10 和 143.99 us/query，HNSW 在 ef=64 时达到 0.9835 recall@10 和 87.36 us/query，体现了近似索引在 latency-recall 权衡上的优势。GPU 实验从公开 DEEP 数据集中使用 HTTP Range 截取前 100000 个 base 向量，在 NVIDIA TITAN Xp 上对 2000 个 query 执行 CUDA Flat ANNS，前 100 个 query 由 CPU 精确搜索校验，得到 recall@10=1.0000，平均 kernel 延迟为 592.37 us/query，含结果拷贝与合并的平均总延迟为 600.40 us/query。本文最终给出 Flat、IVF、PQ、HNSW 与 GPU 扫描在不同并行粒度下的适用边界。

\textbf{关键字：} 近似最近邻搜索；SIMD；OpenMP；乘积量化；CUDA；双调排序
\end{abstract}

\section{引言}

\subsection{研究背景}

深度学习模型通常将文本、图像和音频等非结构化对象编码为高维向量，语义检索问题随之转化为向量空间中的最近邻搜索。给定数据集
\[
X=\{x_1,x_2,\ldots,x_N\}, \quad x_i\in\mathbb{R}^d
\]
以及查询向量 $q$，精确 $k$ 近邻搜索需要返回距离 $q$ 最近的 $k$ 个向量。若直接遍历全部 base 向量并计算距离，单次查询复杂度为 $O(Nd)$。当 $N$ 达到百万级以上时，该成本难以满足低延迟服务需求，因此工程系统通常采用近似最近邻搜索，在召回率和查询延迟之间折中。

ANNS 查询链路中至少包含三类可并行化操作：第一，高维距离计算中的乘加操作，可由 SIMD 指令进行数据级并行；第二，base 向量、倒排桶候选或 query batch 之间相互独立，可由 OpenMP 或 MPI 进行任务级并行；第三，候选集维护和批量扫描具有规则比较-交换结构，可由 GPU 进行大规模线程并行。本文将三次平时作业和期末选题要求整合为一个研究问题：围绕 ANNS 的离线建索引、在线查询和召回评估流程，比较 SIMD、OpenMP、PQ、IVF、HNSW 和 GPU 扫描在不同计算粒度下的性能收益与瓶颈。

\subsection{研究内容与贡献}

本文工作包括以下四部分。

\begin{enumerate}
    \item 梳理 ANNS 相关算法，包括 KD-Tree、LSH、PQ、IVF、HNSW 和 GPU 向量检索系统，明确本文实验与前人工作的关系。
    \item 在 DEEP100K 数据集上实现并比较 Flat Scan、NEON SIMD、OpenMP、SQ-SIMD、PQ-SIMD、PQ-SIMD+OpenMP、IVF-Flat 和 HNSW，分析 latency、recall 和 speedup 的关系。
    \item 实现 CUDA Flat ANNS 查询程序，并在远端 TITAN Xp 服务器完成 GPU 查询实验、真实 DEEP100K 数据下载和 CPU 子集正确性校验。
    \item 按期末选题要求补充完整系统流程：离线预处理包括 PQ 码本训练、IVF 聚类和 HNSW 图构建；在线阶段比较 Flat、IVF、PQ、HNSW 和 GPU 扫描；评估阶段统一使用 CPU 精确 Top-10 作为 ground truth。
    \item 明确区分借鉴内容和本文实现：PQ、IVF、HNSW、GPU Top-k 等算法思想来自已有研究；本文工作在于课程数据集和指定平台上的实现、参数实验、跨并行层次对比与瓶颈分析。
\end{enumerate}

\section{相关工作}

\subsection{近似最近邻搜索算法}

早期最近邻搜索常使用树结构，例如 KD-Tree \cite{bentley1975multidimensional}，但在高维空间中容易受到维度灾难影响。局部敏感哈希（LSH）\cite{gionis1999similarity}通过哈希碰撞概率保持近邻关系，具有理论保证，但通常需要多表和较高内存开销。

乘积量化（Product Quantization, PQ）\cite{jegou2010product}是大规模向量检索中的经典压缩方法。PQ 将高维向量划分为多个子空间，在每个子空间训练码本，并以 code 代替原始浮点向量。查询时通过查表累加完成非对称距离计算（ADC），可以显著降低内存占用和距离计算成本。OPQ \cite{ge2013optimized}进一步通过旋转变换优化子空间划分。IVF \cite{sivic2003video}使用粗聚类建立倒排文件，查询时仅访问最近的若干簇；IVF-PQ 则结合粗筛和 PQ 编码，是 Faiss 等系统中的重要索引结构 \cite{johnson2019billion}。

图索引方法以 HNSW \cite{malkov2018efficient}为代表，通常具有很高召回率和较低延迟，但图遍历分支较多、访存不规则，单次查询内并行化难度高。FastScan \cite{andre2016cache}等研究表明，仅有缓存局部性不足以获得极致性能，还需要面向 SIMD 寄存器和查表结构进行专门设计。

\subsection{GPU 向量检索与候选维护}

向量检索不仅需要计算距离，还需要维护 Top-k 候选。Faiss 的 GPU 研究表明，十亿规模向量检索可以通过 GPU 批量距离计算、倒排桶扫描和高效候选选择获得显著吞吐提升 \cite{johnson2019billion}。GPU 排序研究也表明，规则数据并行算法在大规模输入上可以充分利用 GPU 吞吐和内存带宽 \cite{satish2009sorting}。本文 GPU 部分选择最直接、可验证的精确 Flat 查询作为基线：它不依赖近似索引，能够清楚展示 GPU 对高维向量扫描的加速潜力，也为后续 IVF-PQ GPU 化提供对照。

\section{问题定义与评价指标}

\subsection{距离与召回率}

本文单机 ANNS 实验采用课程 DEEP100K 数据集，base 向量数量为 $100000$，维度为 $96$。距离函数使用内积距离：
\[
dist(q,x)=1-\sum_{i=0}^{d-1}q_i x_i
\]
召回率定义为：
\[
recall@k=\frac{|KNN(q)\cap KANN(q)|}{k}
\]
其中 $KNN(q)$ 为 ground truth 的真实 Top-k，$KANN(q)$ 为算法返回结果。

\subsection{延迟与加速比}

Average latency 表示单次 query 的平均查询延迟，单位为微秒。以串行 Flat Scan 为基准，加速比定义为：
\[
Speedup=\frac{Latency_{baseline}}{Latency_{optimized}}
\]
GPU 排序实验则以 CPU 双调排序为主要基准，同时记录 CPU \texttt{std::sort} 作为串行高效排序参考。

\subsection{系统流程与选题要求对应}

根据期末选题说明，ANNS 报告需要体现完整检索系统、并行方案比较、进阶索引融合和课程学习总结。本文将系统流程划分为三个阶段。离线阶段读取 DEEP100K 数据，构建 PQ 码本与 base code，训练 IVF 粗聚类中心并生成倒排链表，同时构建 HNSW 图索引；在线阶段分别执行 Flat 精确扫描、IVF 倒排桶扫描、PQ ADC 粗排加精排、HNSW 图搜索和 CUDA Flat 扫描；评估阶段使用 CPU 精确 Flat Search 生成 Top-10 参考结果，再计算 recall@10、平均延迟和加速比。

\section{并行算法设计}

\subsection{Flat Scan 与 NEON SIMD}

Flat Scan 遍历全部 base 向量，对每个向量计算一次 96 维内积，并维护 Top-k 优先队列。该方法召回率最高，但计算量为 $O(Nd)$。由于 $d=96$ 能被 4 整除，ARM NEON 可以使用 \texttt{float32x4\_t} 一次处理 4 个单精度数，将 96 次标量乘加转化为 24 次向量乘加。

\begin{lstlisting}[language=C++]
float32x4_t acc = vdupq_n_f32(0.0f);
for (int d = 0; d < 96; d += 4) {
    float32x4_t va = vld1q_f32(base + d);
    float32x4_t vb = vld1q_f32(query + d);
    acc = vfmaq_f32(acc, va, vb);
}
\end{lstlisting}

\subsection{OpenMP 多线程扫描}

对于单个 query，不同 base 向量的距离计算互不依赖。OpenMP 版本将 base 索引划分给多个线程，每个线程维护局部 Top-k 队列，扫描结束后合并局部候选。该设计避免了距离计算阶段对全局队列加锁，适合 Flat Scan 这种负载均匀的任务。

\subsection{SQ-SIMD 与 PQ-SIMD}

SQ 将每一维浮点值线性量化为 uint8，粗排阶段使用更小的数据类型完成近似内积，再对 Top-p 候选使用原始浮点向量 rerank。SQ 的优势是实现简单、访存量低；缺点是逐维线性量化误差较大。

PQ 将 96 维向量划分为 $M$ 个子空间，每个子空间独立训练 $K_s$ 个聚类中心。base 向量离线编码为 $M$ 个 code，查询时先构建 LUT：
\[
LUT[m][c] = dist(q_m, centroid_{m,c})
\]
随后对每个 base code 查表累加：
\[
dist_{PQ}(q,x)=\sum_{m=1}^{M} LUT[m][code_m(x)]
\]
最终仍对 Top-p 候选执行原始浮点 rerank。PQ 的离线训练与编码不计入在线查询延迟。

\subsection{IVF-Flat 与 HNSW 索引}

IVF-Flat 将 base 向量先划分到 $nlist$ 个粗聚类中心。离线阶段使用 KMeans 训练中心，并把每个 base 向量写入距离最近中心对应的倒排链表。在线查询时先计算 query 到所有粗中心的距离，只选择最近的 $nprobe$ 个倒排桶，再在这些桶内执行原始浮点距离计算和 Top-k 维护。$nprobe$ 越大，访问候选越多，recall 越高但 latency 也越大。

HNSW 使用多层小世界图表达近邻关系。离线构建时逐个插入 base 向量，并按参数 $M$ 控制每个节点的邻居数量；在线搜索时从高层向低层贪心下降，在底层维护大小为 $ef$ 的候选队列。$ef$ 越大，搜索越充分，recall 越高但图遍历开销也随之增加。本文使用 hnswlib 的 C++ 头文件实现作为 HNSW 参考实现，并将其与自行实现的 Flat、IVF 和 PQ 路径放在同一 DEEP100K 子集上比较。

\subsection{CUDA 精确 Flat ANNS}

GPU 版本实现精确 Flat ANNS。对于每个 query，CUDA 启动一个线程块，块内 256 个线程按 stride 分片扫描全部 base 向量。每个线程在寄存器中维护自己的局部 Top-10，扫描结束后将局部候选写入全局内存。Host 侧读取每个 query 的 $256\times 10$ 个候选并合并为最终 Top-10。

该设计保持了 Flat Search 的精确性，核心 kernel 只做规则的内积计算和小规模局部 Top-k 插入，避免了复杂索引结构对 GPU 实验的干扰。其并行粒度如下：
\[
q \rightarrow block,\quad base\_id \rightarrow threadIdx.x + t\cdot blockDim.x
\]
核心逻辑如下：

\begin{lstlisting}[language=C++]
int qid = blockIdx.x;
for (int bid = threadIdx.x; bid < base_n; bid += blockDim.x) {
    float ip = 0.0f;
    for (int d = 0; d < dim; ++d) {
        ip += base[bid * dim + d] * query[qid * dim + d];
    }
    insert_local_topk(1.0f - ip, bid);
}
\end{lstlisting}

程序优先读取 \texttt{/anndata} 下的 DEEP100K 数据。本文在 GPU 服务器上通过公开 DEEP 数据源下载 query，并使用 HTTP Range 从 1B base 文件中截取前 100000 个向量，生成 \texttt{DEEP100K.base.100k.fbin}。若数据不存在，程序也保留固定随机种子合成数据的 fallback，便于代码独立复现。

SQ 的粗排速度较快，但 recall 明显偏低，说明简单逐维线性量化不能很好保持原始内积排序。PQ 的效果更稳定：Top-p=2000 时 recall 已达到 0.99630，延迟为 2830.47 us，相比 Flat baseline 加速 7.37 倍；Top-p=10000 时 recall 与 Flat 接近，但延迟也升至 8019.73 us。该结果体现出 PQ 的典型优势：以较小召回损失换取访存压缩和近似距离计算。

\section{综合讨论}

\subsection{不同并行层次的适用边界}

实验结果显示，不同并行技术的收益取决于计算粒度和访存模式。

\begin{itemize}
    \item SIMD 适合固定维度、连续内存、循环结构简单的距离计算。DEEP100K 的 96 维内积正好符合该条件。
    \item OpenMP 适合负载均匀、任务粒度足够大的扫描过程。Flat Scan 和 batch ground truth 生成可以明显受益，但 PQ 单 query 内 ADC 查表粒度过轻，线程管理成本反而更高。
    \item PQ 通过编码压缩降低访存和计算，但需要在 recall 与 latency 之间选择 Top-p 参数。相比 SQ，PQ 的聚类码本更能保留向量空间结构。
    \item IVF 和 HNSW 将“扫描全部 base”改为“访问部分候选”。前者通过 nprobe 控制倒排桶数量，后者通过 ef 控制图搜索宽度，二者都提供清晰的 latency-recall 调节旋钮。
    \item GPU 适合大批量规则向量扫描。精确 Flat ANNS 每个 query 的 base 扫描相互独立，映射到 CUDA block 后能充分利用线程并行；但 partial Top-k 的设备到主机拷贝和 host 合并仍是后续优化点。
\end{itemize}

MPI 更适合在多机环境下按 base 向量分片：每个进程保存一部分 base 或倒排桶，本地完成 Top-k 后再由主进程归并候选。该方案的计算部分与 OpenMP 分片扫描类似，但通信成本主要来自 query 广播和 Top-k 候选汇总。本文实际使用的远端 GPU 环境未提供 MPI runtime，因此没有把 MPI 列入实测主表；报告中保留其分片设计分析，并将可复现实测集中在本机 SIMD/OpenMP、索引参数实验和 CUDA GPU 查询上。

\subsection{前人工作与本文实现的区分}

本文没有提出全新的 ANNS 理论算法。PQ、IVF、HNSW、FastScan 和 GPU 向量检索均来自已有研究。本文的主要工作量在于：第一，围绕课程数据集实现多版本 ANNS 查询算子；第二，在 ARM 平台上对 NEON、OpenMP、SQ/PQ 和调度策略进行对比实验；第三，补充 IVF-Flat 与 HNSW 的索引构建和参数实验；第四，在远端 NVIDIA GPU 上实现并复核 CUDA Flat ANNS 查询；第五，将三次平时作业整理为一个统一的期末研究问题，并给出不同并行层次的适用性分析。

\subsection{复现边界与后续扩展}

本文的可复现实验覆盖了 DEEP100K 上的 Flat、IVF、PQ、HNSW 和 CUDA Flat 查询。GPU 部分采用精确 Flat Search，是为了给出不损失召回的 GPU 扫描基线；近似索引部分则通过 CPU IVF、PQ 和 HNSW 展示候选裁剪带来的 latency-recall 权衡。公开 DEEP 数据源提供的是 1B base 的 ground truth，本文截取前 100K 后使用 CPU 精确搜索生成子集参考结果，因此 recall 评估口径是可复现的，但不是直接复用官方 1B ground truth。后续可在本文代码基础上继续把 IVF 倒排桶扫描、PQ ADC 和 Top-k 合并迁移到设备侧，形成完整 GPU IVF-PQ；也可以把 PQ 查询从 query 内并行改为 query-level batch 并行或线程池复用，以减少轻量查询上的线程调度开销。

\section{结论}

本文围绕近似最近邻搜索核心计算进行了 CPU/GPU 多层次并行优化研究。实验表明，SIMD 与 OpenMP 对 Flat Scan 这种规则高维距离计算具有显著收益，二者结合在 8 线程下获得 27.89 倍加速；PQ-SIMD 能在 Top-p=2000 时保持 0.99630 recall，并获得 7.37 倍加速，优于简单 SQ 量化；PQ 的单 query 内多线程并不总是有效，线程调度和候选合并成本可能超过 ADC 查表收益。进阶索引实验表明，IVF-Flat 在 nprobe=16 时达到 0.9740 recall@10 和 143.99 us/query，HNSW 在 ef=64 时达到 0.9835 recall@10 和 87.36 us/query，二者均能通过参数控制召回和延迟。GPU Flat ANNS 实验表明，在 TITAN Xp 上对 100000 个 96 维 DEEP100K base 向量和 2000 个 query 做精确扫描时，平均 kernel latency 为 592.37 us/query，并在 100 个 query 的 CPU 子集校验中达到 recall@10=1.0000。

综合来看，ANNS 系统优化不能只依赖单一并行手段，而应按计算阶段选择合适层次：距离计算使用 SIMD，粗粒度扫描和 ground truth 生成使用多线程，候选裁剪使用 IVF/HNSW，压缩检索使用 PQ，批量规则向量扫描交给 GPU。本文最终覆盖了题目要求中的完整 ANNS 流程、Flat/IVF/PQ/HNSW 算法比较、SIMD/OpenMP/GPU 并行实验和 latency-recall 分析。
